I. Public Endpoints (No Auth Required)

1. API Status (Home)
```bash
curl -X GET "https://us-central1-products-d644c.cloudfunctions.net/api"
```

2. Get Full Portfolio Data
```bash
curl -X GET "https://us-central1-products-d644c.cloudfunctions.net/api/api/portfolios/john-doe"
```

3. Get Experiences
```bash
curl -X GET "https://us-central1-products-d644c.cloudfunctions.net/api/api/portfolios/john-doe/experiences"
```

4. Get Projects
```bash
curl -X GET "https://us-central1-products-d644c.cloudfunctions.net/api/api/portfolios/john-doe/projects"
```

5. Get Social Links
```bash
curl -X GET "https://us-central1-products-d644c.cloudfunctions.net/api/api/portfolios/john-doe/social"
```

II. Authenticated Endpoints (Firebase ID Token Required)
(Replace <ID_TOKEN> with your Firebase Auth token)

6. Create New Portfolio
```bash
curl -X POST "https://us-central1-products-d644c.cloudfunctions.net/api/api/portfolio" \
     -H "Authorization: Bearer <ID_TOKEN>" \
     -H "Content-Type: application/json" \
     -d '{
       "username": "john-doe",
       "displayName": "John Doe",
       "bio": "Data Scientist & Cloud Architect",
       "photoURL": "https://example.com/photo.jpg",
       "theme": "dark",
       "sectionsOrder": ["experiences", "projects", "socialLinks"]
     }'
```

7. Update Portfolio Metadata & Profile
```bash
curl -X PUT "https://us-central1-products-d644c.cloudfunctions.net/api/api/portfolio" \
     -H "Authorization: Bearer <ID_TOKEN>" \
     -H "Content-Type: application/json" \
     -d '{
       "theme": "light",
       "bio": "Senior NLP Engineer"
     }'
```

8. Add Experience
```bash

     -H "Authorization: Bearer <ID_TOKEN>" \
     -H "Content-Type: application/json" \
     -d '{
       "role": "Data Scientist",
       "company": "Acme Corp",
       "startDate": "2023-01",
       "endDate": null,
       "description": "Designed machine learning pipelines.",
```
